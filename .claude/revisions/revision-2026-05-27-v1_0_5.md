## Fix 1 · Crash khi vào màn hình Cài đặt

**Triệu chứng**: App crash ngay khi mở màn hình Cài đặt (ConfigActivity).

**Root cause**: `SettingsPermissionItem` trong `ConfigScreen.kt` sử dụng:
```kotlin
border = ButtonDefaults.outlinedButtonBorder.copy(
    brush = SolidColor(iconColor.copy(alpha = 0.4f)),
)
```
Trong Compose BOM `2024.09.00` (Compose UI 1.7.x), `BorderStroke` không còn là
`data class` → không có method `.copy()` → `NoSuchMethodError` tại runtime khi
`PermissionsTab` (tab mới thêm v1.0.4) cố render `OutlinedButton`.

**Fix** trong `ConfigScreen.kt`:
- Xóa `ButtonDefaults.outlinedButtonBorder.copy(...)`.
- Thay bằng constructor trực tiếp: `BorderStroke(1.dp, SolidColor(iconColor.copy(alpha = 0.4f)))`.
- Thêm import `androidx.compose.foundation.BorderStroke`.

**Files changed**:
- `ui/config/ConfigScreen.kt` — `SettingsPermissionItem`: thay border expression

## Update 2 · Phát hiện và cài đặt phiên bản mới

**Mô tả**: Khi mở Dashboard, app tự động kiểm tra phiên bản mới nhất từ GitHub
Releases. Nếu phát hiện phiên bản mới hơn phiên bản đang chạy, hiển thị dialog
với thông tin release và hai nút hành động.

**GitHub API**: `https://api.github.com/repos/honghoaibao/Appatpro/releases/latest`

**Hiển thị**:
- Header icon + "Có phiên bản mới" + tên release
- Badge hiển thị tag phiên bản mới (e.g. `v1.0.6`)
- Changelog (`body`) tối đa 300 ký tự, có dấu `…` nếu bị cắt
- Nút **"Để sau"** — dismiss dialog, không nhắc lại trong session
- Nút **"Cập nhật"** — mở link tải APK (từ `assets[0].browser_download_url`
  nếu có, fallback về `html_url`)

**Logic so sánh phiên bản**:
- Lấy `versionName` từ `PackageManager` (e.g. `"v1.0.5"`)
- So sánh với `tag_name` từ GitHub (bỏ qua prefix `v`, case-insensitive)
- Khác nhau → hiển thị dialog; giống → không làm gì
- Lỗi mạng hoặc timeout → không làm gì (silent)

**Files changed / created**:

### `network/UpdateChecker.kt` (mới)
- Object `UpdateChecker` với `suspend fun check(context): UpdateInfo?`
- Data class `UpdateInfo(tagName, releaseName, body, htmlUrl, downloadUrl)`
- Dùng `kotlinx-serialization-json` để parse JSON (đã có trong deps)
- Timeout: connect 6s, read 6s

### `ui/dashboard/DashboardViewModel.kt`
- Import `UpdateChecker`, `UpdateInfo`
- `DashboardUiState` thêm field `updateInfo: UpdateInfo? = null`
- Thêm `fun checkForUpdate()` — gọi `UpdateChecker.check()` trong `viewModelScope`
- Thêm `fun dismissUpdate()` — xóa `updateInfo` khỏi state

### `ui/dashboard/DashboardScreen.kt`
- Import `android.net.Uri`, `UpdateInfo`
- `DashboardScreen` thêm `LaunchedEffect(Unit) { vm.checkForUpdate() }`
- Gọi `UpdateAvailableDialog` khi `state.updateInfo != null`
- Composable `UpdateAvailableDialog(info, onDismiss, onUpdate)` — dialog overlay
  bán trong suốt, card thiết kế nhất quán với `StartupStatusDialog`

## Risks

- `UpdateChecker` dùng `java.net.URL` trực tiếp — không có retry logic.
  Nếu GitHub API down, dialog không hiện (silent fail — chấp nhận được).
- Dialog cập nhật KHÔNG persistent: dismiss xong mở lại app sẽ check lại.
  Có thể thêm SharedPreferences throttle sau nếu cần.
