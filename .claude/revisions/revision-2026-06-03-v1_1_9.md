### 2. Fix lỗi không vuốt qua khi hết giờ gặp bài đăng hình ảnh
**Root cause:** TikTok photo post / image carousel dùng gesture vuốt ngang để chuyển ảnh. Khi engine thực hiện `swipeNext()` (vuốt dọc), TikTok carousel intercept gesture → app đứng yên mà không sang video tiếp theo.

**File:** `NodeTraverser.kt`
- Thêm `detectImagePost(root)` — 3 tier detection:
  - **Tier 1:** Resource-ID fragment scan (`photo_item`, `carousel_dot`, `page_indicator`, v.v.)
  - **Tier 2:** ContentDescription pattern `<N>/<M>` kèm keyword ảnh/photo
  - **Tier 3:** Text soft signals (`"vuốt để xem thêm ảnh"`, `"swipe for more photos"`)

**File:** `AutomationEngine.kt`
- Trước bước Watch: `detectImagePost(root)` → set `isImagePost`
- Watch time: nếu `isImagePost` → cap tại `5_000ms` (không cần xem lâu ảnh)
- Overlay action label: `"🖼 Xem ảnh (Xs)"` thay vì `"▶ Xem video (Xs)"`
- Step [8] Like: skip double-tap nếu `isImagePost` (tránh gesture bị interpret là swipe carousel)
- Step [11] Swipe: `if (isImagePost) forceSwipeNext() else swipeNext()` — swipe 2× mạnh vượt qua carousel

### 5. Tối ưu random hành động nuôi
**File:** `AutomationEngine.kt` — `Human` object

| Hàm | Trước | Sau |
|-----|-------|-----|
| `microPause()` | `Random(60, 220)ms` đều | 95%: `Random(50, 300)ms`; 5%: `Random(800, 2500)ms` (đọc caption) |
| `swipeDuration()` | `base ± Random(-80, 100)` đều | 35%: `+Random(60, 160)` slow; 65%: `+Random(-60, 80)` fast |
| `swipeNext()` | Start `75%`, End `25%` cố định | Start `Random(68%–82%)`, End `Random(18%–30%)` |
| `doLike()` | `microPause()` giữa 2 tap | `microPause()` + cả 2 tap có jitter độc lập + `likeAnimDelay(350–900ms)` |

Hàm mới trong `Human`:
- `swipeStartFactor(): Float` — `[0.68, 0.82]`
- `swipeEndFactor(): Float` — `[0.18, 0.30]`
- `likeAnimDelay()` — `delay(350, 900)ms` sau double-tap

**File:** `ui/dashboard/DashboardScreen.kt` — `StartupStatusDialog` được redesign hoàn toàn:
