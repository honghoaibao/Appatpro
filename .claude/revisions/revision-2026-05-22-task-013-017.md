## Why

Flutter was the original UI framework but became redundant once all 6
screens were migrated to native Compose (TASK-007 through TASK-016).
Keeping Flutter in the build chain added:
- ~40–60s extra CI time (Flutter setup + pub get + analyze)
- APK size overhead (Flutter engine ~4MB)
- `flutter.sdk` path dependency in local.properties (blocks pure Android builds)
- Build complexity: 2 toolchains (Flutter + Gradle) for 0 Flutter UI

`FlutterBridge.sendEvent()` was retained as a stub (not removed) because
5 Kotlin modules call it for LAN broadcast. Removing the stub would require
touching `AutomationEngine`, `PopupHandler`, `TikTokAccessibilityService`,
`LanWebSocketServer`, `OverlayFarmMonitor` in one pass — higher risk,
zero runtime benefit. Deferred safely.

## Rollback

**Rollback to Flutter (full):**
1. `git revert` this commit
2. Restore `lib/`, `pubspec.yaml`, `.flutter-plugins*`, `.metadata`
3. Revert `MainActivity.kt` to extend `FlutterActivity`
4. Revert `FlutterBridge.kt` to full implementation
5. Revert `settings.gradle` / `app/build.gradle` to Flutter plugin versions
6. Revert `build.yml` to Flutter build steps
7. Set all `FeatureFlags.*` = false to restore Flutter routing

**Rollback flags only (keep Flutter removed, revert to old behavior):**
Not possible — Flutter source is deleted. Flags-only rollback would break
routing (no Flutter router to receive events).

**Rollback point:** `atpro_android_BACKUP/` in build environment (session-local).

## Risks

| Risk | Level | Mitigation |
|------|-------|------------|
| `MainActivity` redirect flicker (brief white frame) | LOW | DashboardActivity has dark theme — negligible on modern hardware |
| `FlutterBridge` stub not thread-safe for sendEvent | LOW | `LanWebSocketServer.broadcast()` is already called from multiple threads; existing behavior unchanged |
| `settings.gradle` hardcoded ndkVersion may drift | LOW | NDK 25.1 is stable LTS; update explicitly when upgrading AGP |
| CI version injection via `sed` brittle | LOW | Only touches `versionCode 1` / `versionName "1.0"` — unique strings |
| OEM QA not yet run on physical devices | MEDIUM | Manual QA required before releasing to users (see QA-013-native-screens-checklist.md) |
