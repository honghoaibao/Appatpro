### 4. `.github/workflows/build.yml` — conditional wrapper step

Changed "Generate Gradle Wrapper" (unconditional) to
"Ensure Gradle Wrapper jar (bootstrap if missing)" (conditional):

```yaml
# Before (session 8):
- name: Generate Gradle Wrapper
  working-directory: android
  run: |
    gradle wrapper --gradle-version 8.3 --distribution-type all
    chmod +x gradlew

# After (session 9):
- name: Ensure Gradle Wrapper jar (bootstrap if missing)
  working-directory: android
  run: |
    if [ ! -f gradle/wrapper/gradle-wrapper.jar ]; then
      echo "gradle-wrapper.jar not in repo — generating via system gradle"
      gradle wrapper --gradle-version 8.3 --distribution-type all
    else
      echo "gradle-wrapper.jar present — skipping generation"
    fi
    chmod +x gradlew
```

This is safe for both repo states and becomes a ~1s no-op after TD-CI-001 is resolved.

## Why

**gradlew / gradlew.bat**: These are text files that should always be in version
control for any Android project. They are stable, human-readable, and required
for `./gradlew` to work. No reason not to commit them immediately.

**gradle-wrapper.jar**: Binary JAR that bootstraps the Gradle download. Cannot
be authored manually — requires a Gradle installation to generate. Deferred
to a local run rather than blocking the rest of the CI improvements.

**Conditional step**: Strictly better than the unconditional generate step from
session 8. Same behavior when jar is absent; faster (no-op) when jar is present.
Provides a clean migration path without a hard cutover.

**Phantom dir removal**: `android/{app` was filesystem noise from a template
rendering failure. Removing it prevents confusion for any developer inspecting
the repo, and removes the risk of an editor or build tool accidentally picking
it up in a glob.
