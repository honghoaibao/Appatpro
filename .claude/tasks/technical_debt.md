## TD-CI-001: Gradle wrapper jar not committed to repo

**Severity**: MEDIUM (CI has conditional fallback — no longer blocking)
**Layer**: build
**Module**: `android/`
**Status**: Partial — `gradlew` + `gradlew.bat` committed. `gradle-wrapper.jar` still missing.
**Reason**: `gradle-wrapper.jar` is a binary that must be generated by a local Gradle
installation. The CI workflow now has a conditional bootstrap step:
- If jar is present in repo → CI uses it directly (fast, reproducible)
- If jar is missing → CI generates it via system `gradle` (adds ~15–30s)

**Fix**: Run once locally:
```bash
bash scripts/commit_gradle_wrapper.sh
```
Requires `gradle` installed locally. After running, the CI bootstrap step
becomes a no-op (jar present → skips generation). The step can then be
removed from build.yml to clean up.
**Added**: 2026-05-22
**Updated**: 2026-05-22 (session 9 — gradlew + gradlew.bat now committed)

## TD-BUILD-001: kotlin-kapt DependencyHandler.module() — Gradle 8.x incompatibility
**Severity**: RESOLVED ✅
**Layer**: build
**Module**: `android/`
**Root cause**: `kotlin-kapt:1.8.22` calls `DependencyHandler.module(Object)` internally.
Method removed in Gradle 8.0. Project uses Gradle 8.3 → hard failure.
**Resolution**: Upgraded Kotlin 1.8.22 → 1.9.25 (all 3 plugins + stdlib + Compose Compiler).
**ADR**: ADR-0007
**Resolved**: 2026-05-23 (session 11)

## TD-KSP-001: Migrate Room annotation processing từ kapt → KSP (low priority)

**Severity**: LOW (không blocking)
**Layer**: build
**Module**: `db/`
**Reason**: JetBrains đã deprecated `kotlin-kapt`. KSP (Kotlin Symbol Processing) là
successor chính thức. Room hỗ trợ KSP từ v2.5.0 (project dùng 2.6.1 → fully compatible).
KSP nhanh hơn kapt và tránh hoàn toàn class incompatibilities như TD-BUILD-001.
**Fix scope**: Thay `id "org.jetbrains.kotlin.kapt"` bằng `id "com.google.devtools.ksp"`,
đổi `kapt "androidx.room:room-compiler:..."` thành `ksp "..."`. Cần test Room DAOs sau migration.
**ADR required**: Có — trước khi thực hiện.
**Added**: 2026-05-23
