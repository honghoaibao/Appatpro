## Tính năng mới

### 1. Phát hiện và chuẩn hoá ID tài khoản không hợp lệ trong switch popup

**Vấn đề:** Khi quét danh sách tài khoản từ popup "Chuyển đổi tài khoản" của TikTok, một số entry hiển thị sai do lỗi TikTok:
- **Loại 1 — ID thuần số:** "@6784523189328" (TikTok internal ID bị lộ, không phải @unique_username)
- **Loại 2 — Display name:** "Bảo Hoài1502", "Ảo vãi" (TikTok hiện nickname thay vì @unique_username)

Cả 2 loại đều khiến bước match thiết bị ↔ Golike thất bại.

**Giải pháp (`normalizeInvalidAccountIds`):**
1. `parseAccountListWithNodes()` phân tích popup, đánh flag `isNeedsNormalize = true` cho entry không hợp lệ
2. Với mỗi entry cần chuẩn hoá: click vào node → TikTok switch sang acc đó → Kill + Relaunch → đọc `@username` thực qua `detectCurrentAccount()`
3. Username hợp lệ được thêm vào `discovered` list
4. Sau khi xử lý xong, mở lại popup (để `positionFirstAccount` dùng tiếp)

**Files thay đổi:**
- `NodeTraverser.kt`:
  - Thêm `displayText` computed property cho `NodeResult`
  - Thêm `AccountEntry(displayText, node, isNeedsNormalize)` data class
  - Thêm `isValidTikTokUsername(s)` — false cho: thuần số, chứa Unicode, chứa dấu cách, quá ngắn
  - Thêm `parseAccountListWithNodes(root)` — Pass 1 (@ prefix) + Pass 2 (ASCII regex) + **Pass 3** (Unicode display names)
- `AutomationEngine.kt`:
  - `openSwitchPopup()` → dùng `parseAccountListWithNodes()` thay `parseAccountList()`
  - Thêm `normalizeInvalidAccountIds(invalidEntries)` — fix flow đầy đủ

### 2. Cập nhật Golike API headers theo chuẩn golike.py

`GolikeApi.openConnection()` đã được cập nhật khớp với `_base_headers()` và `_auth_headers()` trong `golike.py`:

| Header | Cũ | Mới |
|--------|----|-----|
| `Accept` | `application/json` | `application/json, text/plain, */*` |
| `Content-Type` | `application/json` | `application/json;charset=utf-8` |
| `Accept-Language` | _(thiếu)_ | `vi-VN,vi;q=0.9,...` |
| `Origin` | _(thiếu)_ | `https://app.golike.net` |
| `Referer` | _(thiếu)_ | `https://app.golike.net/` |
| `User-Agent` | _(thiếu)_ | `Mozilla/5.0 (Linux; Android 10; K)...` |
| `Sec-Ch-Ua` / `Sec-Fetch-*` | _(thiếu)_ | Thêm đầy đủ |
| `G-Auth` | Có | **Xóa** (không có trong golike.py) |

## Lưu ý kỹ thuật

- `isValidTikTokUsername()` là static method của `NodeTraverser` companion — dùng được từ mọi nơi
- `AccountEntry.node` là reference đến `AccessibilityNodeInfo` thực — cần gọi `host.clickNode(entry.node)` ngay trong session để tránh node bị recycle
- Sau `normalizeInvalidAccountIds()`, TikTok đang ở feed của acc invalid cuối cùng — `openSwitchPopup()` sẽ mở lại popup trước khi return
