## 1. Cơ chế Follow mới

**Thay đổi:** Khi farm và gặp video cần follow, thay vì tìm nút Follow trực tiếp trên feed, engine nay:
1. Swipe từ phải sang trái (85%→10% screenWidth, giữa màn hình) → vào trang hồ sơ tác giả
2. Kiểm tra profile bị khoá / riêng tư (`NodeTraverser.detectBlockedProfile`)
3. Tìm nút Follow (`NodeTraverser.findFollowButtonOnProfile`)
4. Click follow → xác nhận nút đổi sang "Following" (`NodeTraverser.isFollowConfirmed`)
5. Back về feed

**Files thay đổi:**
- `AutomationEngine.kt` — `doFollowFromProfile()`, `doFollowOnOpenProfile()`, `FollowResult` enum
- `NodeTraverser.kt` — `findFollowButtonOnProfile()`, `detectBlockedProfile()`, `isFollowConfirmed()`
- Hàm `doFollowWithRoot()` giữ nguyên (dùng trong task mode backup)

## 2. Golike Login qua WebView

**Thay đổi:** Thay form đăng nhập username/password bằng WebView trỏ vào `https://app.golike.net`.

**Flow:**
1. User bấm "Đăng nhập Golike" ở ServicesScreen
2. `DashboardActivity` launch `GolikeLoginWebActivity` qua `ActivityResultLauncher`
3. WebView mở `https://app.golike.net`, user đăng nhập bình thường
4. JS injection sau mỗi page load đọc `localStorage.getItem("token")` và các key phổ biến
5. `TokenBridge.onTokenFound(token)` → `GolikeRepository.saveWebToken(token)` → `RESULT_OK`
6. `DashboardActivity` nhận kết quả → `golikeVm.receiveTokenFromWebLogin(token)` → refresh

**Files mới:**
- `GolikeLoginWebActivity.kt` — WebView + JS bridge + token persistence

**Files thay đổi:**
- `GolikeApi.kt` — BASE_URL: `golike.net` → `app.golike.net`
- `GolikeRepository.kt` — `saveWebToken(token)` public method + `completeTikTokJob(success)` overload
- `GolikeViewModel.kt` — `receiveTokenFromWebLogin(token)` method
- `DashboardActivity.kt` — register `ActivityResultLauncher` + wire callback
- `AndroidManifest.xml` — register `GolikeLoginWebActivity`

## 9. Hotfix — Feed detection & Live detection (patch cùng v1.2.1)

### 9a. Feed drift detection trong watch loop
**Vấn đề:** Engine xem video `videoWatchTimeMax` giây mà KHÔNG kiểm tra có còn ở feed không.
Nếu TikTok tự chuyển sang live room hoặc popup che phủ, engine phải đợi hết watchSecs (có thể 5-10 phút nếu người dùng cài watchMax cao) mới phát hiện.

**Fix — `AutomationEngine.kt`:**
- Thêm constant `FEED_CHECK_INTERVAL_SECS = 5L`
- Trong vòng lặp watch `for (w in watchSecs downTo 1L)`: khi `w % 5 == 0`, gọi `isOnFeedTab()`
- Nếu không ở feed → set `driftedDuringWatch = true` → `break`
- Sau vòng lặp: nếu drift → `continue` outer loop → `isLostWithRetry()` xử lý recovery

### 9b. Inbox tab không tìm thấy → recovery
**Vấn đề:** `doViewInbox()` trả về im lặng khi không tìm thấy tab Hộp thư — có thể do đang ở màn hình sai.

**Fix:**
- Thêm fallback: `pressBack()` → kiểm tra `isOnFeedTab()` → `recoverToFeed()` nếu cần → thử `findInboxTab()` lần 2
- `doViewShop()` giữ nguyên không có recovery (nhiều acc không có tab Cửa hàng — null là hợp lệ)

### 9c. Live detection cải thiện
**Vấn đề kép:**
1. **False positive** của `isOnFeedTab()`: live room có text "bạn bè", "đề xuất" từ live recommendation strip → `isOnFeedTab()` trả true nhầm → `isLostWithRetry()` = false → drift không được phát hiện
2. **False positive/negative** của `detectLive()`: "viewers" đơn lẻ match caption video bình thường; thiếu ID gift-panel của TikTok APK 2025

**Fix — `NodeTraverser.kt`:**

`isOnFeedTab()`:
- Thêm **Tier 0 live-room exclusion**: kiểm tra resource-ID đặc trưng live room (`liveroom`, `gift_entrance`, `live_chat`, `audience_tab`, v.v.) trước mọi keyword check
- Nếu có bất kỳ live-room ID nào → return false ngay

`detectLive()`:
- Thêm **Tier 0**: gift-panel + audience IDs — đặc trưng TUYỆT ĐỐI của live room (`gift_entrance`, `gift_btn`, `audience_tab`, `pk_battle`, `live_viewer_count`)
- Bổ sung Tier 1 IDs cho TikTok APK 2025: `live_screen`, `live_container`, `live_interact`, `gift_list`, v.v.
- Tier 2 strict signals: thêm "phòng live", "đang phát sóng", "thoát khỏi phòng live", "you are now live"
- **Tier 3 (mới)**: viewer count pattern `Regex("""[\d,.]+\s*[kKmM]?\s*(người xem|viewer|viewers)""")` — thay cho "viewers" đơn lẻ
- **Tier 4 soft signals**: LOẠI "viewers" và "live now" (false-positive cao), giữ chỉ tín hiệu gift-specific
